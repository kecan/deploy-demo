## 前置准备

### 1. 阿里云服务器

- 推荐配置：1核2G 或以上
- 操作系统：Ubuntu 20.04/22.04 LTS
- 确保以下端口已开放：
  - 22 (SSH)
  - 80 (HTTP)
  - 443 (HTTPS)
  - 4000 (应用端口，调试用)

> **端口开放位置**：
> - **ECS 服务器**：在「安全组」中添加入方向规则
> - **轻量应用服务器**：在「防火墙」中添加规则

### 2. 本地环境

- Node.js 18+
- Git
- XShell 或其他 SSH 工具
- Xftp 或 scp 命令行工具

### 3. 获取服务器连接信息

- 服务器公网 IP
- 登录用户名（通常是 root 或创建的普通用户）
- 登录密码或 SSH 密钥

## 服务器环境配置

### 方式一：使用初始化脚本（推荐）

将 `scripts/server-init.sh` 上传到服务器并执行：

```bash
# 在服务器上执行
chmod +x server-init.sh
sudo ./server-init.sh
```

### 方式二：手动安装

#### 1. 更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

#### 2. 安装 Node.js 20.x

```bash
# 添加 NodeSource 仓库
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# 安装 Node.js
sudo apt install -y nodejs

# 验证安装
node --version  # 应显示 v20.x.x
npm --version   # 应显示 10.x.x
```

#### 3. 安装 PM2 进程管理器

```bash
sudo npm install -g pm2

# 设置开机自启
pm2 startup
# 执行输出的命令
```

#### 4. 安装 Nginx

```bash
sudo apt install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
```

#### 5. 配置防火墙

```bash
sudo ufw allow ssh
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 4000
sudo ufw enable
```

## 本地打包

### 1. 进入项目目录

```bash
cd deploy-demo
```

### 2. 安装依赖

```bash
# 安装根目录依赖
npm install

# 安装前端依赖
cd client
npm install

# 安装后端依赖
cd ../server
npm install
cd ..
```

### 3. 构建前端

```bash
cd client
npm run build
cd ..
```

构建完成后，会在 `client/dist` 目录生成静态文件。

### 4. 创建部署包

```bash
# 创建临时目录
mkdir -p deploy-package

# 复制前端构建产物（保持 client/dist 目录结构）
mkdir -p deploy-package/client
cp -r client/dist deploy-package/client/

# 复制后端代码（排除 node_modules 和数据）
cp -r server deploy-package/server
rm -rf deploy-package/server/node_modules
rm -rf deploy-package/server/data

# 复制 PM2 配置
cp ecosystem.config.cjs deploy-package/

# 打包
tar -czf deploy-package.tar.gz deploy-package

# 清理临时目录
rm -rf deploy-package
```

## 上传部署

### 方式一：使用 XShell + Xftp

1. **打开 XShell**，连接到服务器
2. **打开 Xftp**（XShell 工具栏中的图标）
3. 将 `deploy-package.tar.gz` 拖拽到服务器的 `/tmp` 目录
4. 在 XShell 中执行以下命令：

```bash
# 创建项目目录
mkdir -p ~/deploy-demo
cd ~/deploy-demo

# 解压部署包
tar -xzf /tmp/deploy-package.tar.gz -C /tmp/
mv /tmp/deploy-package/* ./
rm -rf /tmp/deploy-package

# 安装后端依赖
cd server
npm ci --only=production

# 创建数据目录
mkdir -p data

# 返回项目根目录
cd ..

# 创建日志目录
mkdir -p logs

# 启动应用
pm2 start ecosystem.config.cjs --env production
pm2 save
```

### 方式二：使用 scp 命令（命令行）

```bash
# 在本地执行 - 上传部署包
scp deploy-package.tar.gz your-user@your-server-ip:/tmp/

# SSH 连接到服务器
ssh your-user@your-server-ip

# 后续步骤同上
```

### 方式三：使用部署脚本（推荐）

项目已提供自动化部署脚本：

```bash
# 在本地项目根目录执行
chmod +x scripts/deploy-manual.sh
./scripts/deploy-manual.sh your-user your-server-ip

# 示例
./scripts/deploy-manual.sh root 47.96.123.456
```

## 验证部署

### 1. 检查 PM2 状态

```bash
pm2 status
```

应看到 `deploy-demo` 进程状态为 `online`。

### 2. 查看应用日志

```bash
pm2 logs deploy-demo
```

### 3. 测试 API

```bash
curl http://localhost:4000/api/health
```

应返回：`{"status":"ok","timestamp":"..."}`

### 4. 访问应用

在浏览器中访问：`http://你的服务器IP:4000`

## Nginx 配置

为了更好的性能和安全性，建议使用 Nginx 作为反向代理。

### 1. 复制配置文件

```bash
sudo cp nginx/deploy-demo.conf /etc/nginx/sites-available/deploy-demo.conf
```

### 2. 修改配置

```bash
sudo nano /etc/nginx/sites-available/deploy-demo.conf
```

修改 `server_name` 为你的域名或服务器 IP：

```nginx
server_name your-domain.com;  # 或服务器IP
```

### 3. 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/deploy-demo.conf /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo systemctl reload nginx
```

### 4. 配置 HTTPS（可选但推荐）

使用 Let's Encrypt 免费证书：

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书
sudo certbot --nginx -d your-domain.com

# 证书会自动续期
```

## 常见问题

### Q1: 启动时报错 "EACCES: permission denied"

```bash
# 修复权限
sudo chown -R $USER:$USER ~/deploy-demo
```

### Q2: 端口 4000 被占用

```bash
# 查找占用进程
lsof -i :4000

# 或修改端口
export PORT=3000
pm2 restart deploy-demo
```

### Q3: 数据库文件损坏

```bash
# 备份并删除旧数据库
mv ~/deploy-demo/server/data/todos.db ~/deploy-demo/server/data/todos.db.bak

# 重启应用会自动创建新数据库
pm2 restart deploy-demo
```

### Q4: PM2 进程异常退出

```bash
# 查看错误日志
pm2 logs deploy-demo --err --lines 50

# 重启
pm2 restart deploy-demo
```

### Q5: 更新部署

重复「本地打包」和「上传部署」步骤，或直接使用部署脚本：

```bash
./scripts/deploy-manual.sh your-user your-server-ip
```
